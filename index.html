Chrome Extension

Privacy Policy

This Privacy Policy explains how this Chrome extension collects, uses, stores, and shares information.

Last updated: 17 April 2026

Overview

This extension is designed for authorized users who work in a private support workspace. Its purpose is to help those users monitor saved teams, group them into custom collections, refresh queue snapshots, and review queue health in a browser side panel.

Information This Extension Collects

Depending on how the extension is used, it may collect and store the following categories of information:

  • Workspace content and metadata needed to provide the extension's features, such as saved team identifiers, team names, grouped collections, queue item titles, status values, timestamps, age-band calculations, and related workspace summary data.
  • A work email address, if the user chooses to enter one for the optional feedback and usage-tracking features.
  • Feedback submitted by the user through the extension, including the selected category and the message content the user enters.
  • Limited extension-usage events, such as opening the panel, saving or updating teams, refreshing data, opening a queue, copying summaries, and similar feature-use actions.
  • Local diagnostic log entries generated by extension actions and error handling.

How Information Is Used

The extension uses collected information only to provide and improve its stated functionality. This includes:

  • displaying saved teams and grouped collections in the side panel
  • refreshing and summarizing queue snapshot data
  • highlighting items that are over target or likely to breach target
  • opening the related queue page when requested by the user
  • storing local settings and state between browser sessions
  • receiving bug reports, feature requests, or general feedback submitted by the user
  • measuring feature usage to improve the extension's workflow and reliability

Where Information Is Stored

Most extension state is stored locally in the browser using Chrome extension storage. This can include saved teams, grouped collections, queue snapshots, configuration values, feedback queue entries, usage-event queue entries, and log data.

If the user enables the feedback and usage-tracking features by providing a work email address, certain information may also be sent over HTTPS to a configured Google Apps Script endpoint for internal product support, feedback review, and operational analysis.

Information Sharing

We do not sell user data.

We do not use or transfer user data for advertising, profiling, creditworthiness, lending decisions, or other unrelated purposes.

Information may be shared only in the following limited circumstances:

  • with the configured Google Apps Script endpoint when the user enables optional feedback or usage tracking
  • when necessary to operate, maintain, or improve the extension's single purpose
  • when required to comply with applicable law, regulation, legal process, or enforceable governmental request
  • when necessary to investigate, prevent, or address security, fraud, or abuse issues

Remote Code

The extension does not load or execute remotely hosted JavaScript or WebAssembly code. All executable extension code is packaged with the extension itself. The extension does make HTTPS network requests to retrieve workspace data and to send optional feedback and usage records.

User Choices

  • A work email address is only stored if the user chooses to provide it for feedback and usage tracking.
  • Feedback is only sent when the user actively submits it.
  • If a user stops using the extension or clears extension storage, locally stored data in Chrome storage can be removed.

Data Retention

Information stored locally remains in Chrome extension storage until it is updated, removed by the user, or cleared when the extension is removed.

Information sent to the configured feedback or tracking endpoint may be retained according to the operator's internal retention practices for product support and operational analysis.

Security

The extension is intended to use secure HTTPS connections for supported network requests. Reasonable measures are used to reduce unauthorized access, disclosure, or misuse of stored and transmitted data.

Children's Privacy

This extension is not directed to children.
